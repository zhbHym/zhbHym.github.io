使用IntelliJ IDEA搭建maven+smm框架（一）

2018-11-11   Spring 

创建于2018年3月27日

一 、使用的环境

以下是我使用的环境

  • Intellij IDEA 2017.3.4

  • Tomcat-9.0.5

  • JDK 1.8.0_144

  • Spring 4.2.6

  • MySql 5.6.39

  • Maven -3.5.3

    关于 Intellij IDEA、JDK、MySql 的安装网上教程很多，在此就不再多说。下面其他软件安装。

二、Maven 和 Tomcat 的下载和安装

下载安装Maven

点击 Apache Maven到官网，进入了下载页面，往下拉可发现当前版本是3.5.3，点击下面红框中的 apache-maven-3.5.3-bin.tar.gz 就可下载，下载后解压缩到相应目录下(我的目录是D:\Program Files\maven)

解压后建立系统变量 MAVEN_HOME ，指定 maven 文件夹，我的文件夹为 D:\Program Files\maven\apache-maven-3.5.3

在path中加入以下代码：%MAVEN_HOME%bin

此时还需配置本地仓库,如果不配置，会在C盘自己用户下面的 .m2 文件夹下创建本地仓库，如果配置，会改变为配置的路径。我在 ·· D:\Program Files\maven·· 目录下新建 LocalWarehouse 文件夹，打开 apache-maven-3.5.3 目录中的 conf 文件夹下的 settings.xml 文件夹。在 localRepository标签中加入自己的仓库路径。（我的是 D:/Program Files/maven/LocalWarehouse ）

  • 注意：里面的路径是双斜杠

此时 maven 已经安装完成。还需要在IntelliJ中配置，使所有项目共享 Maven 仓库。
打开IntelliJ,依次打开 File -> settings… -> Build,Execution,Deployment -> Maven ，按照下图所示选择对应的文件夹，home directory选择自己Maven的安装路径，Loacl repository选择自己的仓库位置。

  1. 如上图标注 2 所示，Import Maven projects automatically 表示 IntelliJ IDEA 会实时监控项目的 pom.xml 文件，进行项目变动设置。

  2. 如上图标注 3 所示，在 Maven 导入依赖包的时候是否自动下载源码和文档。默认是没有勾选的，也不建议勾选，原因是这样可以加快项目从外网导入依赖包的速度，如果我们需要源码和文档的时候我们到时候再针对某个依赖包进行联网下载即可。IntelliJ IDEA 支持直接从公网下载源码和文档的。

  3. 上图标注 4 所示，可以设置导入的 VM 参数。一般这个都不需要主动改，除非项目真的导入太慢了我们再增大此参数。

Tomcat 下载与安装

点击 Apache Tomcat进入官网，在左侧Download中选择要下载的版本，然后根据自己的jdk选择对应的版本

解压到对应的目录下（我的目录为 D:\Program Files\apache-tomcat-9.0.5）

然后增加系统变量 CATALINA_HOME

然后在 path 中添加以下路径：%CATALINA_HOME%\bin

此时 Tomcat 已经安装完成，可以在cmd窗口测试，利用快捷键 win + r 打开运行窗口，在里面输入cmd然后回车，进入cmd黑窗口，然后进入 Tomcat 目录下的bin文件夹，输入 startup ,会弹出一个黑窗口，等待它停止输出，然后不要关闭窗口。

在浏览器地址栏输入 http://localhost:8080/ 如果出现如下窗口，则表示安装成功。

三、创建Maven项目

打开 IntelliJ IDEA ,依次打开File -> new -> Project…，打开新建项目窗口，如下图所示

首先选择 Maven，然后选好jdk,选择Maven webapp模型，然后点击 next

填好自己的包名和项目名，然后点 next

在IntelliJ IDEA中默认含有 Maven2 和 Maven3 两个版本，但这里我们选择自己安装的本地 Maven ，然后点击next。

在这个界面中选择好项目位置，直接点击 Finish 完成项目创建。

然后等待项目创建完成，当到IntelliJ 右下角弹出如下窗口时，点击 Enable Auto-Import(自动导入)。

此时需要手动完成包的创建，在 main 目录下创建 java 文件夹，在 src 文件夹下创建 test 文件夹，并在 test 文件夹下创建 java 和 resourse 文件夹。并点击File -> project Structure，

在 Modules 中点击 Job Project ，在右侧的 Sources 在，选中 java ,然后点击Sources属性，依次更改test中的java、resources 属性为 Tests 、 Test Resources 。

基本的配置就完成了，接下来就是代码部分了。

四、配置Tomcat

配置 Tomcat 的方法：首先在工具栏的右侧找打如下图所示的下拉列表，然后点击 Edit Configurations… ，就会弹出Run/Debug Configurations 对话框，如下图所示：

在对话框中点击加号，找到Tomcat Server 选择 Local ，如图所示：

就会添加 Tomcat Server ，更改Name为Tomcat，Application server中选择自己安装的版本，然后点击②，进入 Deployment 中，如下图所示：

进入Deployment页面后点击右侧的 “＋”，选择Artifact：

然后点击 war exploded ,然后依次点击OK ，Finished 完成创建

点击绿色三角形点击运行

如果成功，在浏览器地址栏输入 localhost:8080 回出现如下图所示的的页面：

五、代码配置

如图为创建的目录：

打开 pom.xml 配置文件，里面已经存在一定的配置，但我们要根据所需要的jar包在这里配置。可以在 这里 找到所有的 maven 代码。

在 pom.xml 文件中，我们可以看到这个文件包含了我们之前定义的本项目的gropId等信息，这些信息是该项目的标识，我们不要去改动它们。重点看 <dependencies>标签，翻译过来是”依赖“的意思，也就是说把对每个包的需求都称为一个依赖<depedency>，定义在<dependencies>中。在每个<depedency>中，你需要提供的是所需jar包的groupId、artifactId、version这三个必要信息。比如上面我们看到引入可一个junit包，格式如下：

<dependency>
    <groupId>junit</groupId>
    <artifactId>junit</artifactId>
    <version>3.8.1</version>
    <scope>test</scope>
</dependency>

这是单元测试包，提供了三个基本信息，第4个scope对其他包来说是非必需的。所有jar包的引入都要满足这个格式。

在 <project> 标签中添加如下版本信息，可以自己添加

<properties>
    <!-- 指定编码格式 -->
    <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    <project.reporting.outputEncoding>UTF-8</project.reporting.outputEncoding>

    <!-- 配置相关jar包的版本 -->
    <spring.version>4.3.14.RELEASE</spring.version>
    <fastjson.version>2.9.4</fastjson.version>
    <mybatis.version>3.4.5</mybatis.version>
    <mybatis-spring.version>1.3.1</mybatis-spring.version>
    <commons-dbcp.version>1.4</commons-dbcp.version>
    <commons-codec>1.10</commons-codec>
    <mysql.version>6.0.6</mysql.version>
    <aspectjweaver.version>1.5.3</aspectjweaver.version>
    <poi.version>3.17</poi.version>
    <commons-fileupload.version>1.3.1</commons-fileupload.version>
    <log4j.version>2.10.0</log4j.version>
    <servlet-api.version>4.0.0</servlet-api.version>
</properties>

打开 src\main\webapp\WEB-INF 文件夹下的 web.xml 文件，在 <web-app> 中修改信息为：

<context-param>
    <param-name>log4jConfigLocation</param-name>
    <param-value>classpath:conf/log4j.properties</param-value>
</context-param>
<servlet>
    <display-name>DispatcherServlet</display-name>
    <servlet-name>DispatcherServlet</servlet-name>
    <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
    <init-param>
        <description></description>
        <param-name>contextConfigLocation</param-name>
        <param-value>classpath:conf/spring-*.xml</param-value>
    </init-param>
    <load-on-startup>1</load-on-startup>
</servlet>
<servlet-mapping>
    <servlet-name>DispatcherServlet</servlet-name>
     <url-pattern>*.do</url-pattern>
</servlet-mapping>

在src\main\resources文件夹下新建名为 conf 的文件夹，然后在 conf 文件夹下创建 spring-mvc.xml 文件，创建过程如图所示：

然后在里面添加如下代码：

<!--配置组建扫描-->

</mvc:annotation-driven>

<!--配置MVC注解扫描-->
<context:component-scan base-package="org.smart.controller">

添加后spring-mvc.xml文件的所有代码如图所示:

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"

    <!--配置组建扫描-->
    <mvc:annotation-driven>
    </mvc:annotation-driven>

    <!--配置MVC注解扫描-->
    <context:component-scan base-package="org.smart.controller">
    </context:component-scan>

在 conf 文件夹中新建 log4j.properties 文件，里面输入以下代码：

log4j.rootLogger=INFO,Console,File

#控制台日志
log4j.appender.Console=org.apache.log4j.ConsoleAppender
log4j.appender.Console.Target=System.out
log4j.appender.Console.layout=org.apache.log4j.PatternLayout
log4j.appender.Console.layout.ConversionPattern=[%p][%t][%d{yyyy-MM-dd HH\:mm\:ss}][%C] - %m%n

#普通文件日志
log4j.appender.File=org.apache.log4j.RollingFileAppender
log4j.appender.File.File=logs/ssm.log
log4j.appender.File.MaxFileSize=10MB
#输出日志，如果换成DEBUG表示输出DEBUG以上级别日志
log4j.appender.File.Threshold=ALL
log4j.appender.File.layout=org.apache.log4j.PatternLayout
log4j.appender.File.layout.ConversionPattern=[%p][%t][%d{yyyy-MM-dd HH\:mm\:ss}][%C] - %m%n

在 conf 文件夹中新建 jdbc.properties 文件，里面输入以下代码：

driverClasss=com.mysql.jdbc.Driver
jdbcUrl=jdbc:mysql://localhost:3306/jobproj?serverTimezone=UTC
# 用户名
usernames=root
# 密码
password=

#定义初始连接数
initialSize=0
#定义最大连接数
maxActive=20
#定义最大空闲
maxIdle=20
#定义最小空闲
minIdle=1
#定义最长等待时间
maxWait=60000

在 conf 文件夹中新建 spring-mybatis.xml 文件，里面输入以下代码：

<comtext:property-placeholder location="classpath:conf/jdbc.properties"/>

<!--配置dbcp连接池：连接到数据库-->
<bean id="dataSource" destroy-method="close" class="org.apache.commons.dbcp.BasicDataSource">
    <!--连接池的基本参数-->
    <property name="driverClassName" value="${driverClasss}"></property>
    <property name="url" value="${jdbcUrl}"></property>
    <property name="username" value="${usernames}"></property>
    <property name="password" value="${password}"></property>
    <!--连接池的可选参数-->
    <property name="maxActive" value="${maxActive}"></property> <!-- 最大连接量-->
    <property name="initialSize" value="${initialSize}"></property> <!--初始连接量-->
    <property name="maxIdle" value="${maxIdle}"></property> <!--空闲时保持的连接量-->
    <property name="maxWait" value="${maxWait}"></property>
    <property name="minIdle" value="${minIdle}"></property>
</bean>

<!--配置mybatis的session工厂-->
<bean id="sqlSessionFactory" class="org.mybatis.spring.SqlSessionFactoryBean">
    <property name="dataSource" ref="dataSource"></property>
    <!--申明MyBatis文件保存的地方-->
    <property name="mapperLocations" value="classpath:mapper/*.xml"></property>
</bean>

<!--配置MyBatis的自动扫描接口-->
<bean class="org.mybatis.spring.mapper.MapperScannerConfigurer">
    <!--规定数据访问接口的存储位置-->
    <property name="basePackage" value="org.smart.dao"></property>
</bean>

<!--(事务管理)transaction manager, use JtaTransactionManager for global tx-->
<bean id="transactionManager" class="org.springframework.jdbc.datasource.DataSourceTransactionManager">
    <property name="dataSource" ref="dataSource"></property>
</bean>

<!-- (事务管理)transaction manager, use JtaTransactionManager for global tx -->
<tx:annotation-driven transaction-manager="transactionManager"></tx:annotation-driven>

配置基本完成。
