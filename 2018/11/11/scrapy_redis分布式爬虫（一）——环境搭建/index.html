scrapy_redis分布式爬虫（一）—— 环境的安装测试

2018-11-11   scrapy爬虫 

开发环境

  • IDE采用 Pycharm
  • 数据库采用MYSQL 以及 Navicat
  • 语言使用 Python3
  • 使用虚拟环境 virtualenv

安装Python3、 Pycharm Professional、MySQL比较简单，官网下载安装即可。

安装virtualenv虚拟环境

安装virtualenv的原因：

如果我们要同时开发多个应用程序，那这些应用程序都会共用一个Python，就是安装在系统的Python3。如果应用A需要python3，而应用B需要python怎么办？
这种情况下，每个应用可能需要各自拥有一套“独立”的Python运行环境。virtualenv就是用来为一个应用创建一套“隔离”的Python运行环境。

安装过程：

安装virtualenv虚拟环境比较简单，不用提前下载，直接安装就好。利用快捷键 win+R 打开运行窗口，输入cmd后回车，打开黑窗口，输入以下命令，全局安装virtualenv虚拟环境。所有第三方的包都会被pip安装到Python3的site-packages目录下。

pip install virtualenv

注意： 如果安装比较慢，可以采用豆瓣的镜像安装，但可能得不到最新的版本。安装豆瓣镜像文件只需要在中间加 -i https://pypi.douban.com/simple/即可，即为：

pip install -i https://pypi.douban.com/simple/ virtualenv

此时手动创建文件夹，并创建虚拟环境，
例如我在D:\Project文件夹下创建了名为Evns文件夹，并 cd 到 Evns 文件夹下，利用以下代码就可以创建test虚拟环境。

virtualenv test

也可以创建指定python语言的虚拟环境，使用如下命令：

virtualenv -p python文件路径 虚拟环境名字

python文件路径是指指定python文件的绝对路径，具体到python.exe文件

安装 virtualenvwrapper

virtualenvwrapper是用来管理virtualenv的扩展包，用着很方便。

全局安装即可，会自动检索你安装的Python文件，并安装到lib文件夹下，安装命令如下：

pip install virtualenvwrapper-win

注意：
以前只需安装virtualenvwrapper就可以，但在最近的一次安装中，必须安装virtualenvwrapper-win才可以，安装virtualenvwrapper会找不到命令。

此时就可以使用workon命令了，在cmd下直接输入workon命令，就可以看到工程的名字，如果输入workon + 项目名字直接可以进入项目。测试结果如下图所示：

可以新建虚拟环境，利用命令 mkvirtualenv + 虚拟环境名字，例如创建名为test的虚拟环境：

此时新建的虚拟环境在默认路径，可以修改虚拟环境存放地址，只需要在系统环境变量中创建WORKON_HOME指定路径即可。

创建好了虚拟环境，就可以安装所需要的包，利用workon +虚拟环境的名字进入虚拟环境，创建必要的scrapy包等，例如：

pip install requests/scrapy

如果安装失败，可以到 https://www.lfd.uci.edu/~gohlke/pythonlibs/ 网站下载对应的文件，利用pip命令安装即可。

python -m pip install "Twisted-17.9.0-cp36-cp36m-win32.whl"

此时就可以创建项目了，由于Pycharm无法创建scrapy项目，所以需要在cmd中创建项目，利用如下命令：

scrapy startproject +项目名字
